### How this configuration works:
1. **GitHub** is serving as your code backup/collaboration repository (which is great to keep your code safe and track version history).
2. **Cloudflare Pages** is unaware of GitHub pushes because there is no Git webhook or webhook credentials linked to build your project from GitHub.
3. Because of this, pushing commits to GitHub **does not trigger deployments**.
4. The live sites are updated **exclusively** when a manual deploy is triggered from the command line using the Wrangler CLI (which pushes the local directory directly to Cloudflare Pages):
   * **For Customer site:** `npx wrangler pages deploy . --project-name foreveryoung --branch main --commit-dirty=true`
   * **For Staff site:** `npx wrangler pages deploy . --project-name staff-foreveryoung --branch main --commit-dirty=true`

### How to get automatic deploys from GitHub (Optional)
If you want pushing to GitHub to automatically update the live site, we have two options:
1. **Option A (Cloudflare Git Integration):** We can log into your Cloudflare Dashboard, go to your Pages project settings, and connect it to your GitHub repository. (Cloudflare will handle builds on their servers).
2. **Option B (GitHub Actions):** We can create a GitHub Action workflow file (e.g. `.github/workflows/deploy.yml`) in the repositories, which runs `wrangler pages deploy` automatically every time you push to the `main` branch.
